## 搬瓦工迁移服务是什么

搬瓦工迁移服务（官方叫 Migrate to another DC），就是把你的 VPS 从当前机房物理迁移到另一个数据中心。迁移过程中，VPS 上的所有数据完整保留，停机时间通常是几秒到几分钟，迁完之后 IP 地址会变成新机房的 IP。

简单说：机器位置换了，数据还在，IP 换了。

这个功能适合几种场景：

- 当前机房速度慢或延迟高，想试试别的线路
- 所在地区换一个机房网络效果更好
- 就是想换个 IP（但不是被封的情况，这点后面会讲）

## 迁移前必须知道的四件事

**第一，数据不会丢。** 这是官方明确说明的——迁移过程中 VPS 上所有数据都被保留，你不用在迁移前重装系统或者清空数据。不过官方也建议迁之前先做个备份，以防万一。搬瓦工本身有免费的自动备份功能，迁移前跑一下确认一下最新备份存在，心里踏实。

**第二，IP 会变。** 换到新机房就是新 IP，这是迁移的连带效果。如果你的 VPS 上跑着网站，域名解析要对应更新成新 IP，DNS 生效需要几分钟到几小时不等，建议在流量少的时候操作。如果服务配置文件里有绑定旧 IP 的地方，也要一并改掉。

**第三，切回去 IP 大概率能恢复。** 比如你从 DC2 迁到 DC8，之后再迁回 DC2，原来的 IP 大概率还在，除非这段时间里原 IP 被别人分配走了。不需要特别担心这个问题。

**第四，迁移会消耗流量。** 免费、次数不限，但每次迁移本身产生的数据传输会计入你当月的流量额度。消耗多少跟 VPS 上的数据量有关，一般不多，但流量本来就快超的时候要注意一下时机。

## 搬瓦工迁移服务操作步骤

整个流程很短，基本上 3 步就完成了。

1. **登录搬瓦工官网后台**，进入 Services → My Services，找到你的 VPS 实例，点击旁边的 **KiwiVM Control Panel** 按钮，进入 KiwiVM 控制面板。

2. **找到迁移功能**。在 KiwiVM 左侧菜单里，找到 **Migrate to another DC** 这一项，点进去。页面会显示你当前的机房位置，以及可以迁移的目标机房列表。

3. **选机房，确认，等待**。选好目标机房，点 **Confirm on next step**，进入确认页面后点 **Start Migration**，就开始了。迁移时间根据 VPS 上数据量从几分钟到几十分钟不等，完成后会收到系统发的邮件通知，里面包含新的 IP 地址。

迁移期间 VPS 基本可用，但可能有短暂的几秒钟中断。

## 迁移失败怎么处理

迁移时最常见的两种报错：

**错误 734152**（Migration backend is currently not available）：大多数情况下是 VPS 的 IP 被封了，KiwiVM 后端无法与服务器通信，所以迁移命令下不去。不是系统故障，根源是 IP 的问题。

排查方法：在 KiwiVM 面板里找 IP 检测功能，或者进入 `kiwivm.64clouds.com/[VPS_ID]/main-exec.php?mode=blacklistcheck` 测试 IP 状态。如果显示 **IP BLOCKED**，那迁移这条路就走不通了，需要换 IP。

**错误 734104/734105**（目标机房暂时无可用 IP / 区域已满）：这是目标机房临时资源不足，一般等几个小时再试，或者换一个别的目标机房。

**IP 被封后的正确处理方式**：直接走付费换 IP 流程，在 KiwiVM 面板里找到换 IP 的申请入口，价格按申请页面为准，通常是人工处理，等待时间在 24 小时以内。这条路才是真正的解决方案，不要在迁移上来回试浪费时间。

## 迁移机房之后要做什么

迁完不是结束，有几个地方要跟着改：

1. **更新域名解析**：把旧 IP 换成新 IP，操作在你的域名注册商那边。DNS 生效有延迟，可以先用新 IP 直接测试服务是否正常。

2. **检查服务配置**：如果跑着 Nginx、Apache 或者其他绑定了 IP 的服务，配置文件里的旧 IP 要改成新 IP，不然服务起不来。

3. **更新 SSH 配置**：本地 `~/.ssh/config` 或者 PuTTY 里保存的主机地址，换成新 IP 才能继续连上去。

4. **通知相关方**：如果这台机器上的服务对接了外部 API、Webhook 或者白名单，要把新 IP 告知对方并更新。

## 常见问题 FAQ

**搬瓦工迁移机房要花钱吗？**

不收费。迁移本身是免费功能，次数不限。唯一的代价是每次迁移会消耗一定的月流量额度，具体消耗量跟 VPS 上的数据总量有关。

**迁移之后数据会丢失吗？**

不会。官方明确说明迁移过程中 VPS 上所有数据完整保留，停机时间在几秒到几分钟之间。迁完之后数据还在原来的位置，不需要重新配置。

**迁移机房能不能实现换 IP？**

可以，迁到新机房会分配新 IP。但有一个前提：当前 IP 必须是正常可用状态。如果 IP 已经被封，迁移功能会被系统禁用，这时候只能走付费换 IP 的流程。

**我怎么知道哪个机房更适合我？**

最直接的方式是买 CN2 GIA-E 套餐，然后把几个主要机房都切一遍，对比实际连接速度和延迟。搬瓦工各机房都有测试 IP 可以提前 ping 一下，大概判断延迟水平再决定往哪里迁。

**迁移需要多长时间？**

一般 15 分钟以内完成，具体取决于 VPS 上数据量的大小。迁完会收到邮件通知，邮件里包含新机房的 IP 地址。如果等了很长时间还没动静，可以重新进 KiwiVM 面板查看当前机房是否已经变更。

**香港套餐可以迁到别的机房吗？**

不可以。香港 CN2 GIA 套餐比较特殊，不支持机房迁移，也不能从其他套餐迁入。买了就是香港，适合对延迟要求极高且预算充裕的用户。
